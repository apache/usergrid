    Making Queries

    Making Queries

    Use the Query String field to enter SQL-style queries against your collections. For example, to select all users whose name starts with fred:

    select * where name = 'fred*'

    To select all activities where a category is usermessage and content contains the word hello in a string:

    select * where category = 'usermessage' and content contains 'hello'
    Learn more about Queries and Parameters.
    ×

    Query Path

    Query Path

    The query path is typically just the name of the collection you want to access. For example, if you want to work with a dogs collection, then your path will be: 

    /dogs

    You may also have a more complex path, such as the case when you want to make a connection between two entities. To create a likes connection between a user named Fred and a dog named Dino, do a POST operation to: 

    /users/fred/likes/dogs/dino
    Learn more about Entity Relationships.
    ×

    Limit

    Limit

    The limit parameter is used to tell the API how many results you want to have returned from the API call.

    The default setting is 10.

    The max setting is 999.

    This parameter is appended to the end of the query string to be sent to the API. For example, a limit of 100:

    GET /dogs?limit=100
    Learn more about Queries and Parameters.
    ×

    JSON

    JSON

    The JSON is the object notation used to describe your object.

    This parameter makes up the body that is sent with POST and PUT requests

    {
            "uuid":"00000000-0000-0000-000000000000",
            "type":"book",
            "title":"Great Expectations"
          }
    Learn more about using the API.
    ×

    Method

    Method

    The Method is used to tell the API what type of operation you want to perform. These http methods map to the standard CRUD methods:

    POST is used for CREATE

    GET is used for READ

    PUT is used for UPDATE

    DELETE is used for DELETE

    Using these four methods, you can perform any type of operation against the API. For example, to READ from a collection called /dogs:

    GET /dogs
    Learn more about using the API.

    Apple Push Notification Service
    A Notifier allows App Services to connect to and deliver a message to a communication provider such as Apple's APNs. Upload Development and Production Certificates (.p12) to set up a bridge between your app and APNs for push notifications on iOS devices.
    For more help: view our getting started page for more info on how to generate and download an iOS .p12 certificate at the Apple Developer Connection website.
    The notifier name is used as the key for push data. Give this a name that describes the certificate being uploaded.



    Only applicable if your certificate is password protected
